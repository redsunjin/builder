# AI Builder Product Roadmap

본 로드맵은 **AI Builder라는 '프로덕트(UI 목업 조립 서비스)'**가 사용자에게 어떠한 가치를 제공하며 성장해 나갈 것인지에 대한 비즈니스/기능적 마일스톤입니다.

---

## Phase 1: MVP (Alpha) - 컴포넌트 조립 기능 증명 (현재 완료)
**목표:** '최소한의 목업 자동 생성 경험 증명' 및 '토큰 절약' 가능성 수치화 확인.

*   [x] 사전 정의된 기본 컴포넌트(헤더, 네비게이션, 영웅 섹션, 푸터 등) 5~10종 확보 및 캐싱 시스템 구축.
*   [x] 사용자의 1줄 텍스트 입력만으로, 백그라운드 엔진이 해당 컴포넌트들을 호출하여 하나의 단순 HTML 페이지(단일 화면) 렌더링.
*   [x] 토큰 절감률(Telemetry) 측정 대시보드 데모 성공.

---

## Phase 2: PoC 고도화 (Beta) - 시장성 및 효율성 극대화 (현재 집중 단계)
**목표:** 경쟁 AI 빌더 서비스(예: v0.dev, Open Claw) 대비 자사 엔진의 독창성/속도 차별화 증명.

*   [ ] **하이브리드 Multi-LLM 적용:** 
    *   요구사항 파싱 등 직관적인 일은 비용이 들지 않는 로컬 모델(Ollama)이 처리, 미려한 UI 코드가 필요한 동적 컴포넌트는 고성능 모델(OpenAI/Gemini)이 처리하도록 라우터 결합.
*   [ ] **일반인용 웹 UI 제공:**
    *   현재 CLI/백엔드 로직 중심에서 벗어나, 사용자가 플라스크 웹앱(`web/app.py`) 상에서 실시간 채팅을 통해 피드백을 주면 화면이 리렌더링되는 시각적 채팅창 도입.
*   [ ] **컴포넌트 카탈로그 확장:** 
    *   초기 10종의 컴포넌트에서, e-Commerce, SaaS Dashboards, Landing Pages 등 도메인별 50종 이상의 다채로운 "저비용/캐시 응답용" 컴포넌트 사전 구축 및 크롤링 확보.

---

## Phase 3: 확장 및 베타 서비스 런칭 (Gamma)
**목표:** 다중 화면 지원 및 유려한 UX 제공 등 사용성 확보의 궤도 오름.

*   [ ] **멀티 화면 (Routing) 지원:** 단일 페이지 HTML 생성을 넘어, 네비게이션을 클릭하면 "About", "Contact" 등 여러 페이지가 연결된 구조의 미니 웹사이트 세트 자동 생성.
*   [ ] **사용자 시각 편집기 연동:** AI가 조립해준 화면에서 색상이나 패딩 여백, 폰트 등을 노코드(No-code) 방식의 우측 사이드 툴바를 통해 미세 조정하는 기능.
*   [ ] **내보내기 (Export) 다각화:** ZIP 파일 압축 다운로드 (HTML+CSS+Assets), React / Vue.js 포맷 변환 및 Next.js 템플릿 직행 추출 기능 지원.

---

## Phase 4: 수익화 및 엔터프라이즈 Live (Final)
**목표:** 정식 B2B/B2C 비즈니스 궤도로 스케일업 및 유료화.

*   [ ] **계정 연동 및 구독 플랜:** 사용자 프로필별 생성 히스토리 격리 저장, 한도 토큰(Credit) 결제 모델 확립.
*   [ ] **Figma / 외부 호스팅 익스텐션 배포:** Vercel 원클릭 배포 연동 및 생성물을 디자이너 친화적인 포맷으로 연계.
*   [ ] **AI Component Marketplace (생태계 구축):** 사용자들이 자신이 만든 멋진 커스텀 원자 컴포넌트 파츠를 AI 캐시에 등록하고 수익을 공유하는 마켓플레이스화.
